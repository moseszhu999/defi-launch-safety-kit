# GitHub Marketing Playbook

This repository is both a code asset and a technical trust asset.

## Positioning

> A lightweight pre-audit and launch-readiness toolkit for DeFi/token projects.

## Funnel role

```text
LinkedIn / X / Discord / Email / GitHub Issues
        ↓
GitHub repository
        ↓
README + demo report + docs + releases
        ↓
Short technical call / pre-audit review / launch-readiness service
```

## What the repository must prove

1. The tool solves a real launch-readiness problem.
2. It is defensive and report-oriented.
3. It can be installed and tested quickly.
4. It produces useful reports.
5. The maintainer can provide professional pre-audit preparation support.

## Landing page checklist

- Clear one-line headline
- Short "Who this is for" section
- Clear "What it checks" section
- Quick demo command
- Sample output list
- Service positioning section
- Disclaimer that this is not a formal audit
- Links to structured import, readiness, baseline, and policy docs
- Repository description and topics configured
- Repository pinned on the maintainer profile

## 30-day content loop

### Week 1

- Polish README
- Add launch-review one-pager
- Add LinkedIn launch post
- Pin the repo

### Week 2

Publish one post:

> I built a lightweight DeFi/token launch-readiness toolkit.

### Week 3

Publish one technical article:

> 20 risks to check before launching an ERC20 token.

### Week 4

Create one public case study using a demo contract.

## Key principle

Do not market this as a full audit. Market it as pre-audit preparation and launch-readiness triage.
