# v1.2 Notes — Structured solc AST and Slither JSON Imports

v1.2 adds structured import support so DeFi Launch Safety Kit can consume compiler and analyzer outputs instead of relying only on source-code heuristics.

## Added

- `dlsk import-struct` command
- `--solc-json` option for `dlsk scan`
- `--slither-json` option for `dlsk scan`
- solc Standard JSON AST import
- Slither JSON import
- compiler-derived contract surface map
- imported structured findings with stable source metadata
- structured import metadata in Markdown, HTML, JSON, and SARIF reports

## Example

```bash
dlsk import-struct \
  --solc-json examples/structured/solc-standard-json-output.json \
  --slither-json examples/structured/slither-report.json \
  --out reports/structured-import
```

```bash
dlsk scan \
  --source examples/contracts/RiskyToken.sol \
  --profile erc20 \
  --solc-json examples/structured/solc-standard-json-output.json \
  --slither-json examples/structured/slither-report.json \
  --format all \
  --out reports/demo_v12 \
  --no-slither
```

## Why it matters

The tool can now prefer compiler-derived structure where available:

- contracts
- inheritance hints
- public/external function surfaces
- modifiers
- mutability
- privileged functions

Slither JSON findings are also imported into the normal DLSK report flow.

This moves the project from pure heuristic scanning toward a more audit-prep-oriented workflow.
