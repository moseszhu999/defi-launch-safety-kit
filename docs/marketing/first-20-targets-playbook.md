# First 20 Targets Playbook

This playbook is for the first 14-day marketing experiment. The goal is not mass outreach. The goal is to get real feedback from small DeFi/token teams.

---

## Target profile

Look for projects that match at least three of these signals:

- public GitHub repository
- Solidity contracts are visible
- ERC20 / staking / vesting / airdrop / treasury / launchpad logic
- recent commits in the last 3 months
- no obvious audit report in the README
- preparing testnet, beta, launch or hackathon demo
- small team where a lightweight readiness review may be useful
- clear docs or Discord/Telegram contact

Avoid:

- anonymous pump tokens
- projects promising yield or price appreciation
- gambling, scam, fake RWA or suspicious fundraising projects
- projects with no public code
- projects asking for backdoors, bypasses or stealth launch help

## 14-day experiment

### Day 1-2

Polish GitHub landing page and pin the repo.

Post the first LinkedIn update.

### Day 3-5

Collect 20 candidate projects.

Do not message everyone immediately. Filter first.

### Day 6-10

Send 5 to 8 highly targeted messages.

Offer a quick readiness pass only for public repos.

### Day 11-14

Try to produce one of these:

- a short feedback note from a team
- an anonymized demo case study
- an improved report format based on feedback
- one small paid review offer

---

## Success metrics

Do not measure only stars.

Measure:

- replies
- GitHub profile visits
- issue openings
- requests for quick review
- LinkedIn connection requests
- project teams sharing a repo or contract address
- one testimonial or anonymized case study

---

## Positioning reminder

Do not sell yourself as a replacement for formal auditors.

Position as:

> Practical launch-readiness support for small DeFi/token teams before formal audit or mainnet launch.

This is more credible and easier to buy.
