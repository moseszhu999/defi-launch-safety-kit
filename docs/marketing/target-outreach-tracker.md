# Target Outreach Tracker

Use this file with issue `#3 Collect first 20 target projects`.

The goal is to find 20 credible small DeFi/token projects, filter them, and contact only the best-fit teams with useful launch-readiness feedback.

Do not spam. Do not publicly accuse projects of vulnerabilities. Do not claim to provide a formal audit.

---

## Target qualification rules

A project is a good candidate if it matches at least four of these signals:

- public GitHub repository
- Solidity contracts are visible
- ERC20, staking, vesting, airdrop, treasury, launchpad, NFT marketplace, or small DeFi logic
- recent commits within the last 30-90 days
- README looks like a real project, not only a toy exercise
- no obvious formal audit report linked in README
- has owner/admin/mint/pause/blacklist/tax/fee/upgrade/deployment risk surface
- has tests, deployment scripts, or CI gaps worth documenting
- project is preparing testnet, beta, launch, hackathon demo, or public release
- has a GitHub issue/discussion/contact channel

Avoid:

- anonymous pump tokens
- projects promising yield or price appreciation
- gambling, scam, fake RWA, or suspicious fundraising projects
- projects with no public code
- projects asking for stealth launch, bypasses, or backdoors
- dead repos with no activity and no team signal

## Outreach sequence

### Step 1: shortlist only

Do not send a message yet. First fill at least 20 candidates.

### Step 2: run a quick public-repo pass

For the top 5 projects, run the public code through DLSK and prepare only 2-3 useful observations.

Focus on documentation and readiness gaps, not public vulnerability accusations.

### Step 3: send a soft message

Use this short version:

```text
Hi — I’m building DeFi Launch Safety Kit, a lightweight AI-assisted pre-audit launch-readiness workflow for small DeFi/token teams.

I’m testing it on public repos and your project looks like a possible fit.

This is not a formal audit and I’m not trying to publicly report vulnerabilities. I’m looking at practical readiness items such as owner/admin permission notes, contract surface map, deployment checklist, CI/test readiness, and audit-prep questions.

If useful, I can share a short launch-readiness note based only on the public repo.
```

### Step 4: record outcome

Track:

- message sent date
- channel used
- reply / no reply
- whether they asked for the report
- what wording got the best response
- whether a case study or testimonial is possible

---

## Success standard

The first campaign succeeds if it produces any one of these:

- 3 meaningful replies
- 1 team asks for a quick readiness note
- 1 founder connects on LinkedIn
- 1 issue/discussion conversation opens
- 1 anonymized case study can be written
- 1 paid pre-audit readiness offer becomes plausible
